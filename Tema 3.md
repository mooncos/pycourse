# Listas

<aside>
üêç Son secuencias ordenadas de datos. Pueden ser datos de cualquier tipo: strings, integers, floats, booleanos, listas, objetos, etc. Son tipos de datos mutables.

</aside>

Las listas son **mutables, ordenadas** y admite **duplicados.**

La mutabilidad se refiere a la capacidad de cambiar, agregar o eliminar elementos despu√©s de que la lista ha sido creada. 

La ordenaci√≥n hace referencia a que las listas en Python mantienen el orden de los elementos seg√∫n el orden en el que fueron agregados. 

Y que admita duplicados significa que los datos pueden estar repetidos dentro de una lista.

```python
lista_1 = ["C", "C++", "Python", "Java"]
lista_2 = ["PHP". "SQL", "React", "Angular"]
```

Podemos acceder a los elementos de una lista a trav√©s de sus √≠ndices [inicio : fin : paso]. El primer elemento no tiene √≠ndice 1, tiene √≠ndice 0.

```python
print(lista_1[1:3])
>> ["C++", "Python"]
```

Podemos ver la cantidad de elementos con la propiedad **len()**

```python
print(len(lista_2))
>> 4
```

Podemos hacer concatenaci√≥n de varias listas con el s√≠mbolo **+**

```python
print(lista_1 + lista_2)
>> ['C', 'C++', 'Python', 'Java', 'PHP', 'SQL', 'React', 'Angular']
```

Vamos a ver diferentes funciones con el siguiente ejemplo

```python
lista_1 = ["JavaScript", "Python", "Django", "Git"]
lista_2 = ["SQL", "PHP", "Visual Studio", "Eclipse"]
lista_3 = ["d", "z", "f", "a", "j"]
lista_4 = [100, 24, 42, -1, 0]
```

Para **agregar** un elemento a una lista en el √∫ltimo lugar se usa **append()**

```python
lista_1.append("Perl")
print(lista_1)
>> ["JavaScript", "Python", "Django", "Git", "Perl"]
```

Para **ordenar** los elementos de una lista en su lugar, es decir, modifica la lista original reorganizando sus elementos de manera ascendente (por defecto) o descendente, se usa el m√©todo **sort()**. Este m√©todo no devuelve un nuevo objeto, solo ordena la lista existente.

```python
lista_3.sort()
print(lista_3)
>> ["a", "d", "f", "j", "z"]
```

Para **eliminar** un elemento de la lista con su √≠ndice usamos **pop()**. Devuelve el elemento que se ha eliminado

```python
print(lista_1.pop(4))
>> "Perl"
```

Podemos **invertir** el orden de los elementos en el lugar con **reverse()**. ‚ö†No es lo opuesto a **sort()**

```python
lista_4.reverse()
print(lista_4)
>> [0, -1, 42, 24, 100]
```

Podemos **insertar** un elemento en una posici√≥n espec√≠fica con **insert()**

```python
lista_1.insert(3, "Ruby") # Ruby ocupar√° el √≠ndice 3.
print(lista_1)
>> ["JavaScript", "Python", "Django", "Ruby", "Git"]
```

Para **eliminar** el primer elemento  con el valor especificado se usa **remove()**

```python
lista_2.remove("PHP")
print(lista_2)
>> ["SQL", "Visual Studio", "Eclipse"]
```

Con **index()** devolvemos la posici√≥n del primer elemento con el valor especificado

```python
ocurrencia = lista_3.index("f")
print(ocurrencia)
>> 2
```

Con **count()** devolvemos la cantidad de veces que aparece un elemento en una lista

```python
ocurrencia = lista_4.count(0)
print(ocurrencia)
>> 1
```

Esto son solo algunos m√©todos de listas que tenemos en Python. Si quieres ver m√°s m√©todos puedes acudir a la documentaci√≥n oficial de Python a trav√©s del enlace: 

[5. Estructuras de datos](https://docs.python.org/es/3/tutorial/datastructures.html)

# index()

<aside>
üêç Usamos el m√©todo index() para explorar strings, ya que permite hallar el √≠ndice de aparici√≥n de un caracter o cadena de caracteres dentro de un texto dado.

</aside>

```python
string.index(value, start, end)
# Sienddo string una variable que almacena un string
# siendo value el caracter o los caracteres buscados
# start es el √≠ndice desde el que se comienza a buscar
# end es el √≠ndice hasta el cual realiza la b√∫squeda. No se incluye.

print(cadena.index("l", 3)

```

Con **rindex()** hacemos lo mismo pero la b√∫squeda es en sentido inverso

```python
string.rindex(value, start, end)

cadena = "hola hola caracola"
print(cadena.rindex("l")
>> 16
```

Tanto el **start** como el **end** son opcionales.

Adem√°s, podemos devolver un caracter en el √≠ndice que le indiquemos al string

```python
string[i]
# Siendo i el √≠ndice

cadena ="hola hola caracola"
print(cadena[5])
> h
```

# Strings: Propiedades

Cuando trabajes con strings en Python, debes tener presente lo siguiente:

1. **Son inmutables:** Una vez creados, no pueden modificarse sus partes, pero s√≠ pueden reasignarse los valores de las variables a trav√©s de m√©todos de strings.

```python
# Si hacemos lo siguiente nos dar√° un error
cadena = "Hola"

cadena[0] = "h"

>> TypeError: 'str' object does not support item assignment
```

1. **Son concatenables:** Es posible unir strings con el s√≠mbolo +

```python
cadena_1 = "Hola"
cadena_2 = "Python"
concatenacion = cadena_1 + " " + cadena_2

print(concatenacion)
>> Hola Python
```

1. **Son mutiplicables:** Es posible concatenar repeticiones de un string con el s√≠mbolo *

```python
cadena_1 = "Hola "
print(cadena_1 * 3)
>> Hola Hola Hola
```

1. **Son multilineales:** Pueden escribirse en varias l√≠neas al encerrarse entre triples comillas simples (‚Äô‚Äô‚Äô ‚Äò‚Äô‚Äô) o dobles (‚Äù‚Äù‚Äù ‚Äú‚Äù‚Äù)

```python
cadena_multilinea_doble = """Este es un ejemplo
con triple comilla doble
para crear una cadena de varias l√≠neas."""
print(cadena_multilinea_doble)
>> Este es un ejemplo
con triple comilla doble
para crear una cadena de varias l√≠neas.

# Tambi√©n puede hacerse con las comillas simples

cadena_multilinea_doble = '''Este es un ejemplo
con triple comilla doble
para crear una cadena de varias l√≠neas.'''
print(cadena_multilinea_doble)
>> Este es un ejemplo
con triple comilla doble
para crear una cadena de varias l√≠neas.
```

1. **Se puede determinar su longitud:** a trav√©s de la funci√≥n **len()**

```python
cadena = "Esta es mi cadena de texto"
print(len(cadena))
>> 26
```

1. **Se puede verificar su contenido:** A trav√©s de las palabras claves **in** y **not in**. El resultado de esta verificaci√≥n es un booleano (True / False)

```python
# Uso de in
cadena = "Hola Python"
resultado = "Python" in cadena
print(resultado)
>> True

# Uso de not in
cadena = "Hola Python"
resultado = "JavaScript" in cadena
print(resultado)
>> False
```

# Strings: M√©todos

Podemos hacer distinciones entre los m√©todos de strings

## M√©todos de an√°lisis

**count():** Retorna el **n√∫mero de veces que se repite** un conjunto de caracteres especificado.

```python
resultado = "Hola Python".count("Hola")
print(resultado)
>> 1
```

**find()** e **index():** Retornan la **ubicaci√≥n** (comenzando desde cero) en la que se encuentra el argumento indicado. La diferencia es que **index** lanza un **ValueError** cuando el argumento no es encontrado, mientras **find** retorna **-1**

```python
resultado = "Hola Python".find("JavaScript")
print(resultado)
>> -1

# Recordad que es case sensitive, es decir, es sensible a may√∫sculas
resultado = "Hola Python".find("python")
print(resultado)
>> -1 # Pues estamos buscando python con la p en min√∫sculas.
```

**rfind()** y **rindex():** Busca el conjunto de caracteres pero **desde el final.**

```python
resultado = "C:/python36/python/python.exe".rfind("/")
print(resultado)
>> 18
```

**startswith( )** y **endswith( )** indican si la cadena en cuesti√≥n comienza o termina con el conjunto de caracteres pasados como argumento, y retornan True o False en funci√≥n de ello.

```python
resultado = "Hola Python".startswith("Hola")
print(resultado)
>> True

# Con endswith()
resultado = "Hola Python".endswith("Hola")
print(resultado)
>> False
```

```python
resultado = "abc123".isdigit()
print(resultado)
>> False
```

```python
resultado = "‚Ö§".isnumeric() # 5 en romano. No es la letra V
print(resultado)
>> True
```

**isdecimal():** Determina si todos los caracteres de la cadena son decimales; es decir, formados por d√≠gitos del 0 al 9.

```python
resultado = "1234".isdecimal()
```

### M√©todos de transformaci√≥n

<aside>
üêç Como los strings son inmutables, los m√©todos a continuaci√≥n no act√∫an sobre el objeto original, sino que retornan uno nuevo.

</aside>

**capitalize():** Retorna la cadena con su primera letra en may√∫scula

```python
# Si hacemos lo siguiente
cadena = "hola Python"
cadena.capitalize()

>> hola Python # Al ser inmutable no se modifica la cadena.

resultado = cadena.capitalize()
```

**encode():** Codifica la cadena con el mapa de caracteres especificado y retorna una instancia del tipo bytes

```python
cadena = "hola Python"
resultado = cadena.encode("utf-8")

>> b'hola Python'
```

**replace():** Reemplaza una cadena por otra

```python
cadena = "Hola JavaScript"
resultado = cadena.replace("JavaScript", "Python")

>> Hola Python
```

**lower():** Retorna una copia de la cadena con todas sus letras en min√∫sculas

```python
cadena = "HOLA PYTHON"
resultado = cadena.lower()
print(resultado)
>> hola python
```

**upper():** Retorna una copia de la cadena con todas sus letras en may√∫sculas

```python
cadena = "hola python"
resultado = cadena.upper()
print(resultado)
>> HOLA PYTHON
```

**swapcase():** Cambia las may√∫sculas por min√∫sculas y viceversa

```python
cadena = "Hola Python"
resultado = cadena.swapcase()
print(resultado)
>> hOLA pYTHON
```

strip( ), lstrip( ) y rstrip( ): remueven los espacios en blanco que preceden y/o suceden a la cadena.

```python
cadena = " hola Python "
resultado = cadena.strip()
print(resultado)
>> 'hola Python' # Usamos las comillas para que se vea la diferencia sin espacios
```

**center(), ljust(), rjust():** Alinean una cadena en el centro, la izquierda o la derecha. Un segundo argumento indica con qu√© caracteres se deben llenar los espacios vac√≠os, que por defecto es un espacio en blanco

```python
cadena = "Python"

```

### M√©todos de separaci√≥n y uni√≥n

**split():** Divide una cadena seg√∫n un caracter separador (por defecto son espacios en blanco). Un segundo argumento indica cu√°l es el m√°ximo de divisiones que puede tener lugar (-1 por defecto para representar una cantidad ilimitada). Devuelve una lista de subcadenas.

```python
cadena = "Hola Pyhton\nHola Python"
resultado = cadena.split()
print(resultado)
>> ['Hola', 'Pyhton', 'Hola', 'Python']

# Veamos otro ejemplo
	cadena = "Hola, c√≥mo est√°s, hermano?"
	resultado = cadena.split(',')
	print(resultado)
>> ['Hola', ' c√≥mo est√°s', ' hermano?']

# Y otro m√°s
cadena = "2022-02-04"
resultado = cadena.split("-")
print(resultado)
>> ['2022', '02', '04']
```

**splitlines( ):** divide una cadena con cada aparici√≥n de un salto de l√≠nea. Devuelve una lista de subcadenas.

```python
cadena = "Hola \nPython"
resultado = cadena.splitlines()
print(resultado)
>> ['Hola ', 'Python']
```

**partition( ):** retorna una tupla de tres elementos: el bloque de caracteres anterior a la primera ocurrencia del separador, el separador mismo, y el bloque posterior.

```python
cadena = "Hola Python. Hola JavaScript"
resultado = cadena.partition(" ")
print(resultado)
>> ('Hola', ' ', 'Python. Hola JavaScript')
```

**rpartition( ):** opera del mismo modo que el anterior, pero partiendo de derecha a izquierda.

```python
cadena = "Hola Python. Hola JavaScript"
resultado = cadena.rpartition(" ")
print(resultado)
>> ('Hola Python. Hola', ' ', 'JavaScript')
```

**join( ):** debe ser llamado desde una cadena que act√∫a como separador para unir dentro de una misma cadena resultante los elementos de una lista. Devuelve los elementos en una sola cadena.

```python
cadena = ", "
resultado = cadena.join(["C", "Python", "Java", "JavaScript"])
print(resultado)
>> C, Python, Java, JavaScript
```

# Tuplas

<aside>
üêç O *tuples,* son estructuras de datos que almacenan m√∫ltiples elementos en una √∫nica variable. Se caracterizan por ser ordenadas e inmutables. Esta caracter√≠stica las hace m√°s eficientes en memoria y a prueba de da√±os.

</aside>

```python
mi_tupla = (1, "dos", [12.56, "ocho"], (5.8, 12))
print(mi_tupla)
>> (1, 'dos', [12.56, 'ocho'], (5.8, 12))
```

Podemos acceder a sus datos. **Indexado**

```python
print(mi_tupla[2][0])
>> 12.56
```

Podemos hacerle **casting** (conversi√≥n de tipos de datos)

```python
lista = list(mi_tupla)
print(lista)
>> [1, 'dos', [12.56, 'ocho'], (5.8, 12)] # Ahora ser√≠a una lista, por tanto, mutable
```

Se le pueden extraer datos (unpacking)

```python
a, b, c, d = mi_tupla
print(a)
>> 1
```

# Diccionarios

<aside>
üêç Son estructuras de datos que almacenan informaci√≥n en pares clave:valor. Son √∫tiles para guardar y recuperar informaci√≥n a partir de los nombres de las claves. No usan √≠ndices

</aside>

```python
mi_diccionario = {"curso":"Python", "clase":"Diccionarios"}
```

Podemos agregarle nuevos datos, o modificarlos

```python
mi_diccionario["recursos"] = ["apuntes", "ejercicios"]
print(mi_diccionario)
>> {'curso': 'Python', 'clase': 'Diccionarios', 'recursos': ['apuntes', 'ejercicios']}
```

Podemos acceder a sus valores a trav√©s del nombre de las claves:

```python
print(mi_diccionario["recursos"][1])
>> ejercicios
```

Con los m√©todos keys(), values(), items(), podemos listar los nombres de las claves, los valores y pares clave:valor

```python
# Keys()
diccionario = {'a': 1, 'b': 2, 'c': 3}
claves = diccionario.keys()
print(claves)
>> dict_keys(['a', 'b', 'c'])

# Values()
diccionario = {'a': 1, 'b': 2, 'c': 3}
valores = diccionario.values()
print(valores)
>> dict_values([1, 2, 3])
# Items()
diccionario = {'a': 1, 'b': 2, 'c': 3}
elementos = diccionario.items()
print(elementos)
>> dict_items([('a', 1), ('b', 2), ('c', 3)])
```

<aside>
üêç A partir de Python 3.7+, los diccionarios son tipos de datos ordenados, en el sentido que dicho orden se mantiene seg√∫n su orden se inserci√≥n.

</aside>

# Sets y m√©todos

<aside>
üêç Son otro tipo de estructuras de datos. Se diferencian de listas, tuplas y diccionarios porque son una colecci√≥n mutable de elementos inmutables, no ordenados y sin datos repetidos.

</aside>

```python
set_1 = {1, 2, "tres"}
set_2 = {3, "tres"}
```

### M√©todos

Podemos agregar un elemento al set con **add()**

```python
set_1.add(5)
print(set_1)
>> {1, 2, 5, 'tres'} o {1, 2, "tres", 5}
```

Con el m√©todo **clear()** removemos todos los elementos de un set:

```python
set_1.clear()
print(set_1)
>> set()
```

Podemos crear una copia del set con **copy()**

```python
set_3 = set_1.copy()
print(set_3)
>> {1, 2, 'tres'}
```

Con **difference(set)** retornamos el set formado por todos los elementos que √∫nicamente existen en el set primero

```python
set_3 = set_1.difference(set_2)
print(set_3)
>> {1, 2}
```

Tambi√©n est√° **difference_update(set)** que remueve del primero todos los elementos comunes al 1 y 2

```python
set_1.difference_update(set_2)
print(set_1)
>> {1, 2}

```

Con **discard(item)** removemos un elemento del set

```python
set_1.discard("tres")
print(set_1)
>> {1, 2}
```

**intersection(set)** retorna el set formado por todos los elementos que existen en el 1 y en el 2 simult√°neamente

```python
set_3 = set_1.intersection(set_2)
print(set_3)
>> {'tres'}
```

**intersection_update(set):** mantiene √∫nicamente los elementos comunes al set 1 y set 2

```python
set_2.intersection_update(set_1)
print(set_2)
>> {'tres'}
```

Con **isdisjoint(set)** devolvemos True si el set 1 y el set 2 no tienen elementos en com√∫n

```python
conjunto = set_1.isdisjoint(set_2)
print(conjunto)
>> False
```

Tambi√©n podemos devolver True **con** issubset(set) si todos los elementos del set 2 est√°n presentes en el set 1

```python
es_subset = set_2.issubset(set_1)
print(es_subset)
>> False
```

Y con **issuperset(set)** devolvemos True si el set 1 contiene todos los elementos del set 2

```python
es_superset = set_1.issuperset(set_2)
print(es_superset)
>> False
```

**pop()** elimina y retorna un elemento al azar del set

```python
aleatorio = set_1.pop()
print(aleatorio)
>> {1} # Si ejecutas de nuevo te puede devolver otro
```

Con **remove(item)** eliminamos un elemento del set, y si no existe arroja un error

```python
set_1.remove("tres")
print(set_1)
>> {1, 2}
```

**symmetric_difference(set)** retorna todos los elementos del set 1 y set 2, excepto aquellos que son comunes a los dos

```python
set_3 = set_2.symmetric_difference(set_1)
print(set_3)
>> {1, 2, 3}
```

Con **symmetric_difference_update(set)** eliminamos los elementos comunes al set 1 y set 2, agregando los que no est√°n presentes en ambos a la vez

```python
set_2.symmetric_difference_update(set_1)
print(set_2)
>> {1, 2, 3}
```

**union(set)** retorna un set resultado de combinar el set 1 y el set 2, eliminando los datos duplicados

```python
set_3 = set_1.union(set_2)
print(set_3)
>> {1, 2, 'tres', 3}
```

**update(set)** inserta en el set 1 los elementos del set 2

```python
set_1.update(set_2)
print(set_1)
>> {1, 2, 3, 'tres'}
```
